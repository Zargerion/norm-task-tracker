# Функционал по разделам

## Раздел "Проекты"

### Отображение
- Список/колонка карточек проектов
- Карточка: иконка проекта, название, цветная полоска (цвет проекта)
- При наведении: красивый tooltip с названием и описанием (Markdown rendered)
- Таймлайн-полоска: горизонтальная шкала с точками milestone

### Создание/редактирование проекта
- Название (обязательно)
- Описание (Markdown, необязательно)
- Иконка (upload файла, image)
- Цвет (выбор из GENSHIN_COLORS)
- Milestones: добавить/удалить точки (дата + описание)
- Участники: добавить/удалить пользователей пространства

### Milestone уведомления
- Cron: каждые 5 мин проверять `Milestone.date <= now && notified=false`
- Отправить TG-уведомление всем `ProjectMember` с TG-аккаунтом
- Проставить `notified=true`
- Текст: "🗓 Важная дата в проекте {название}: {milestone.title}\n{milestone.description}"

## Раздел "Пользователи"

### Иерархическая пирамида
```
        [ Admin1 ]  [ Admin2 ]
    [ Manager1 ] [ Manager2 ] [ Manager3 ]
[ Emp1 ] [ Emp2 ] [ Emp3 ] [ Emp4 ] [ Emp5 ]
```
- Карточка пользователя: аватар (первые буквы имени), имя, должность, любимый цвет (кружок)
- Hover: полная информация (email, phone, description)

### Действия ADMIN
- Изменить роль участника (выпадающий список)
- Редактировать описание
- Удалить участника из пространства

## Раздел "Трекеры"

### Структура
- Селект проекта вверху (с цветом и иконкой проекта)
- 4 колонки (kanban):
  1. CREATED — "Создано"
  2. ACCEPTED — "Принято"
  3. COMPLETED — "Завершено"
  4. APPROVED — "Одобрено"

### Карточка задачи (в списке)
- Цветная левая рамка по сложности (estimatedHours)
- Название задачи
- При hover: шкала статусов (4 точки), заявленное время (левый верх)
- Аватары назначенных юзеров

### Карточка задачи (открытая, modal/page)
- Полное описание (Markdown rendered, с кликабельными ссылками)
- Статус + кнопки перехода (по правам)
- Заявленное время / Реальное время (сумма TaskTimeRecord)
- Назначенные пользователи (добавить/удалить)
- Прикреплённые файлы: картинки — отображать, файлы — кнопка скачать
- Дополнения (комментарии): лента снизу, каждый может добавить Markdown-блок

### TG-уведомления для задач
| Событие | Кому | Что |
|---------|------|-----|
| Назначен исполнитель | Назначенному | Название, описание, ссылка на задачу |
| COMPLETED (завершена) | Всем MANAGER/ADMIN проекта | Название, описание, ссылка, Google Meet + Телемост |
| COMPLETED→ACCEPTED (откат) | Исполнителям задачи | Название, кто откатил, ссылка, Google Meet + Телемост |

### Ссылки в уведомлениях
- Google Meet: https://meet.google.com/new
- Яндекс Телемост: https://telemost.yandex.ru

## Раздел "Пространства" (только SUPER_ADMIN)

- Список всех пространств
- Создать пространство (название, описание)
- Войти в пространство (переключиться)
- Добавить первого ADMIN в пространство (выбрать из списка всех users)
- Удалить пространство

## Раздел "Материалы"

### Типы материалов
1. **Markdown**: вставить текст → отображается красиво
2. **HTML**: загрузить HTML-файл → отображается как HTML

### Поля материала
- Название
- Тип (Markdown / HTML)
- Тело (Markdown-текст) или HTML-файл (upload)
- Прикреплённые файлы (любые)

### Доступ к материалу
- `isPublic = true` → доступен всем, даже без авторизации
- `isSpaceWide = true` → всем участникам пространства
- `MaterialAccess` записи → точечный доступ по userId
- Логика: `isPublic OR isSpaceWide OR (userId IN MaterialAccess)`

### Скачивание
- Markdown: конвертировать в HTML и скачать как `.html`
- HTML: скачать загруженный файл напрямую

## Регистрационная форма (публичная)

|------|:-----------:|
| Имя | ✅ |
| Фамилия | ✅ |
| Логин | ✅ |
| Пароль | ✅ |
| Должность | ❌ |

| Описание | ❌ |
| Телефон | ❌ |
| Email | ❌ |

- Показать UUID-код
- Показать ссылку на TG-бот: `https://t.me/<BOT_USERNAME>?start=<CODE>`
